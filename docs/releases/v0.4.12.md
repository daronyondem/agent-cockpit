# Agent Cockpit v0.4.12

## Shipped For Users

- Fixed the Windows welcome checklist so npm and PM2 no longer show as missing when Agent Cockpit was installed with the private Node.js runtime.

## Developer Details

- Updated `InstallDoctorService` to resolve npm and npx from `install.nodeRuntime.binDir` before falling back to `PATH`, including Windows `npm.cmd` and `npx.cmd` probes.
- Updated npm-backed install-doctor actions so they use the installer-recorded Node runtime when present.
- Added regression coverage for Windows private-runtime npm/PM2 checks in `test/installDoctorService.test.ts`.
- Extended the Windows release smoke installer and static workflow coverage to call `/api/chat/install/doctor` and require `node`, `npm`, and `pm2` to be ready after a private-runtime install.
- Updated specs and release workflow docs for the private-runtime install-doctor behavior.
