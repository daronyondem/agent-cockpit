# Agent Cockpit v0.4.36

## Shipped For Users

- Windows installs and repair runs now show an explicit extraction step and avoid the PowerShell archive progress path that could make the installer appear stuck until the terminal received keyboard focus.
- The Windows installer keeps using the same release ZIP and private Node ZIP verification, but extraction now runs through a non-interactive built-in .NET ZIP API.

## Developer Details

- Added `Expand-ZipFile` to `scripts/install-windows.ps1` and replaced both private Node runtime and app release `Expand-Archive` calls with `System.IO.Compression.ZipFile.ExtractToDirectory`.
- Updated Windows installer static coverage to assert non-interactive ZIP extraction and guard against reintroducing `Expand-Archive` in the user-facing installer.
- Updated deployment/testing specs, project memory, and agent guidance to keep Windows installer extraction non-interactive.
- Mobile PWA impact: none. The change is limited to Windows installer ZIP extraction before dependency install/startup.
