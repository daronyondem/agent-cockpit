# Agent Cockpit v0.4.38

## Shipped For Users

- Windows repair installs now fix stale Codex terminal wrappers, so rerunning `install-windows.ps1 -Repair` replaces npm-generated shims that fail with `node.exe is not recognized`.
- Windows installs now use a raw readiness probe while waiting for Agent Cockpit to answer, avoiding PowerShell web-page parsing prompts that could make the installer appear stuck until the terminal received focus or keyboard input.

## Developer Details

- Added `Repair-CliToolWrappers` to `scripts/install-windows.ps1`. During configure/repair, the installer rewrites existing Agent Cockpit-managed `codex.ps1`/`codex.cmd` wrappers to call the resolved Node runtime plus `@openai/codex/bin/codex.js`, and rewrites Claude wrappers to call `claude.exe` directly when the package is present.
- Replaced the installer `/auth/setup` readiness loop's `Invoke-WebRequest` probe with `System.Net.HttpWebRequest`, preserving the 2-second per-attempt timeout and redirect handling while avoiding HTML parsing and PowerShell security prompts.
- Updated Windows installer static coverage, deployment/testing specs, project memory, and agent guidance. Mobile PWA impact: none; changes are limited to the Windows installer.
