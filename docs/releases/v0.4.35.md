# Agent Cockpit v0.4.35

## Shipped For Users

- Windows production updates now verify that the newly installed Agent Cockpit version is the one actually running before the update is accepted. This prevents an older background server from continuing to serve the app after a newer release has been installed.
- Windows update rollback is more reliable when the new release cannot take over: Agent Cockpit restores the previous install manifest and restarts the previous release through its own local PM2 command.

## Developer Details

- Changed the Windows production restart script generated by `UpdateService` to call the target release's app-local `node_modules\.bin\pm2.cmd` directly instead of `npx pm2`.
- Added version-matched Windows health validation: update restart scripts parse `/api/chat/version` and require the returned `version` to equal the target release version before saving PM2 state.
- Kept the old running Windows process from changing its in-memory local version before the restart handoff, so a failed handoff does not hide that an update is still available.
- Preserved the macOS restart/update path and covered it with focused macOS installer and release packaging regression tests.
- Added ADR-0066 and updated deployment, backend-service, testing, and agent-project-memory docs for the Windows update handoff rule.
