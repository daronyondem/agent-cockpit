# Agent Cockpit v0.4.27

## Shipped For Users

- Fixed Windows Claude Code and Codex setup so the welcome screen can detect CLIs installed by Agent Cockpit immediately after installation.
- Fixed Windows chat startup for welcome-installed Claude Code and Codex profiles by launching the installed package entrypoints directly instead of relying on fragile npm command shims.
- Kept the Windows fix isolated from macOS so the existing Mac installer and CLI launch behavior continue to use the direct command path.

## Developer Details

- Added a shared Windows CLI runtime resolver for Claude Code and Codex. It prefers installer-managed package entrypoints under `<installDir>\cli-tools` and `%APPDATA%\npm`, runs Codex as `node.exe <codex.js>`, runs Claude through the package `claude.exe`, and only falls back to `.cmd` shims through `cmd.exe`.
- Routed Claude Code, Claude Code Interactive compatibility probes, Codex app-server/exec, setup-auth checks, setup-auth login jobs, Codex plan usage, Install Doctor, and CLI update checks through the shared resolver.
- Updated Windows CLI update detection so welcome-installed Codex targets are recognized from the package path and updated with `npm --prefix <installDir>\cli-tools i -g ...` instead of relying on global npm locations.
- Updated specs, project memory, AGENTS guidance, README, and focused tests for the Windows package-entrypoint path. No mobile PWA behavior changed.
