# Agent Cockpit v0.4.20

## Shipped For Users

- Fixed Windows installs so the Agent Cockpit server no longer leaves a Node.js console window visible on the desktop.

## Developer Details

- Changed the Windows PM2 ecosystem config to monitor `bin\run-agent-cockpit.vbs` through `wscript.exe //B //NoLogo` instead of spawning `node.exe` directly.
- The runner now starts the server hidden with `node --import tsx server.ts` and redirects stdout/stderr to install-local runner logs, avoiding the extra `tsx` CLI process layer while keeping PM2 as the owner of restart/save behavior.
- Windows installer and generated start-helper launches now delete any stale `agent-cockpit` PM2 entry before starting from the regenerated hidden runner config.
- Added Windows runner generation to both fresh installer runs and production self-update config rewrites so updated installs keep the hidden-process launch path.
- Added PM2 diagnostics when installer readiness times out so Windows smoke failures include process status and recent logs.
- Kept PM2 ownership, install-local `PM2_HOME`, scheduled-task startup, and private Node runtime behavior unchanged.
- Updated Windows installer/update tests and deployment/testing specs for the hidden runner path.
