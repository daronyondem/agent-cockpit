# Agent Cockpit v0.4.154

## Shipped For Users

- Claude Code profiles can now target Anthropic or AWS Bedrock. Bedrock profiles preserve configured inference-profile IDs/ARNs in the composer while Agent Cockpit still uses the base Claude model metadata for effort, media capability, and usage-cost mapping.
- Codex model selection and usage estimates now match the current Codex CLI catalog, including GPT-5.6 Sol, GPT-5.6 Terra, GPT-5.6 Luna, GPT-5.5, GPT-5.4, GPT-5.4-Mini, and the text-only Codex Spark preview. GPT-5.6 standard and Fast/Priority usage estimates use the current OpenAI rates.
- Mobile chat code blocks are easier to read and reuse. Long code lines wrap within the chat panel, and mobile code blocks include a Copy control.
- Mobile queued sends are more reliable when attaching to an already-running conversation stream; queued messages now drain against the live active conversation instead of stale state.
- The desktop Files browser can preview and edit `.astro` files as text.
- Kiro model metadata now includes Claude Opus 4.8.

## Developer Details

- Added Claude Code provider routing on `CliProfile.claudeCode.provider`, structured Bedrock inference-profile settings, metadata projection from Bedrock base models, composer preservation during metadata loading, plan-usage suppression for Bedrock profiles, usage-pricing pattern coverage, ADR-0090, and updated API/backend/data-model/frontend/testing specs. See #454.
- Added current Codex fallback model metadata, shared `ultra` effort validation across contracts/settings/routines/Workspace Context/mobile reconciliation, OpenAI GPT-5.6 standard and priority pricing entries with cache-write rates, and Spark no-price behavior because the CLI marks it as non-API. See #476.
- Added Kiro Opus 4.8 to the pinned Kiro model catalog with image capability metadata, focused backend tests, and backend/research spec updates. See #450.
- Added mobile Markdown code-block chrome, per-block copy behavior, wrapped desktop/mobile code block styling, and static frontend coverage. See #449.
- Added server and desktop file-preview classification for `.astro` as editable text, `text/x-astro` MIME inference, preview/save regression coverage, and an updated Settings screen bundle budget. See #459.
- Fixed mobile queued-send draining by resolving sends from the live active conversation ref and guarding drains by expected conversation id, with mobile regression coverage. See #466.
- Release prep refreshed root and mobile dependency locks, pinned both Vite ranges to the patched `^8.0.16` floor, fixed Vite hash normalization in the web bundle-budget helper, and raised the main/total JS bundle ceilings to the patched build baseline so `npm audit`, mobile audit, and release budget checks are clean before publishing.
