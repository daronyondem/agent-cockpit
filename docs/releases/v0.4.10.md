# Agent Cockpit v0.4.10

## Shipped For Users

- Fixed the Windows installer on Windows PowerShell 5.1. Fresh installs now generate first-run setup secrets using a PowerShell 5.1-compatible cryptography API instead of failing with `RandomNumberGenerator.Fill` missing.
- Kept the GitHub Release page easier to read by publishing the shipped bullet list directly, without the internal release-document section headings.

## Developer Details

- Replaced `RandomNumberGenerator.Fill($buffer)` in `scripts/install-windows.ps1` with `RandomNumberGenerator.Create().GetBytes($buffer)` plus disposal, preserving cryptographic randomness while supporting the .NET Framework runtime used by Windows PowerShell 5.1.
- Added a regression assertion in `test/windowsInstallerScript.test.ts` so the Windows installer script does not reintroduce the unsupported static `Fill(...)` API.
- Updated `docs/spec-deployment.md` to document the Windows PowerShell 5.1-compatible secret generation path.
- Updated `scripts/render-release-notes.js`, `test/releaseNotes.test.ts`, `docs/release-workflow.md`, and `docs/releases/README.md` so GitHub Release bodies publish a simple shipped bullet list and a full-details link while the source-controlled release document keeps its structured sections.
