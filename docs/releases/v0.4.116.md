# Agent Cockpit v0.4.116

## Shipped For Users

- Workspaces can now be archived from Workspace Settings. Users can retire a workspace while keeping its conversation history, optionally save a verified file snapshot, restore the workspace later, remap a missing folder, or delete the retained archived data from Settings.
- Workspace Context now has dedicated References and Assets areas. Reusable markdown/text references and safe non-executable files can live beside synthesized context, be previewed from Workspace Settings, and be linked from chat on desktop and mobile.
- Workspace Context maintenance now folds accepted active Memory entries into canonical context and removes them from the Memory inbox after validation. The old Memory Review screen and scheduler have been retired in favor of that maintenance path.
- The Workspace Context material browser is easier to scan. Context, References, and Assets now use the same two-pane layout, with list/search on the left and preview or actions on the right.
- Mobile chat follows the latest turn more reliably after sending. Normal sends opt back into tail-following before the optimistic message renders, and **Back to end** repeats the end scroll across layout frames so iOS viewport changes do not leave the transcript above the newest message.
- Claude Code Interactive has been validated with Claude Code 2.1.158, so compatibility warnings and update checks reflect the newest tested CLI boundary.

## Developer Details

- Added archived-workspace metadata, history-only archival, optional verified ZIP snapshots, default restore destinations, original-folder cleanup tracking, and retained-data deletion. `WorkspaceArchiveStore` owns archive summary/lifecycle persistence, `WorkspaceSnapshotService` owns snapshot creation/checksum verification/safe extraction, and `workspaceArchiveRoutes.ts` exposes the route surface. ADR-0080 records the archive/snapshot model. See #388.
- Added desktop archive management in the workspace Archive tab and global Settings **Archived Workspaces** section, with specs covering archive data models, routes, frontend behavior, data layout, mobile scope, and tests. Archive management remains desktop-only in this release.
- Replaced the old Memory Review workflow with Workspace Context maintenance consumption of accepted active Memory inbox entries. Removed the scheduler/UI/MCP review path, kept compatibility `410` responses for former review routes, and updated project guidance/specs to document Memory-as-inbox and Workspace Context-as-canonical-context semantics. ADR-0081 records the ownership change. See #390.
- Added Workspace Context `references/` and `assets/` material directories, server-side allowlists and validation, route handlers, processor prompt inputs, managed instruction updates, desktop Settings management, chat file-link previews, and mobile link handling. ADR-0082 records the material model. See #392.
- Aligned Workspace Context Context/References/Assets UI around a common two-pane browser pattern and moved reference creation/editing plus asset upload/preview/download controls into the preview pane. Asset upload and download actions use primary styling. See #394 and #395.
- Updated mobile transcript scroll helpers and mobile chat rendering so send and **Back to end** flows explicitly re-scroll to the computed transcript end after viewport/composer layout settles. See #393.
- Advanced `CLAUDE_CODE_INTERACTIVE_TESTED_CLI_VERSION` to `2.1.158` after real terminal and browser compatibility reports passed, and updated backend/API specs plus CLI update warning coverage. See #396 and #391.
- Updated API, backend-service, data-model, frontend, mobile PWA, Workspace Context, testing, data-layout, user Memory, ADR, project-memory, and project-guidance docs for the shipped behavior.
- Mobile PWA impact was evaluated across the release. Runtime mobile behavior changed for transcript tail-following and Workspace Context reference/asset link previews; archive management remains a desktop-only management surface.
