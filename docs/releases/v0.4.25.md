# Agent Cockpit v0.4.25

## Shipped For Users

- Fixed Windows welcome-screen installs for Claude Code and Codex so the Install buttons put the CLIs in an Agent Cockpit-owned per-user location and the refreshed welcome checks can detect them immediately.
- Improved Windows CLI startup behavior so Agent Cockpit can launch the installed `claude` and `codex` commands from the managed install path after setup, repair, reinstall, or self-update.

## Developer Details

- Windows Install Doctor now installs Claude Code and Codex with `npm --prefix <installDir>\cli-tools i -g ...`, prefers `<installDir>\cli-tools` during CLI probing, keeps the recorded runtime probe for compatibility, and falls back to `%APPDATA%\npm` before inherited `PATH`.
- Successful Windows Claude/Codex welcome install actions now prepend `<installDir>\cli-tools` to the live server `PATH` before returning the refreshed doctor payload, which keeps immediate setup-auth checks aligned with the newly installed shims.
- The Windows installer and Windows production self-update config now persist `PATH` with `<installDir>\cli-tools`, the private Node runtime when present, `%APPDATA%\npm`, and the inherited user path while preserving macOS runtime-path behavior.
- Specs, project memory, and focused tests were updated for the Windows-only CLI prefix and detection order. No mobile PWA behavior changed.
