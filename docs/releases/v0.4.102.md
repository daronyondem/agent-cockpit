# Agent Cockpit v0.4.102

## Shipped For Users

- Workspace Context links in chat are now directly usable on desktop and mobile. Markdown references open in the file previewer, render as Markdown, and can be downloaded from the same conversation-scoped link.
- Conversations are more resilient when a local CLI session id disappears. Agent Cockpit can preserve the prior discussion as a recovery snapshot, show a clear recovery notice, and continue in a fresh native session for Claude Code, Codex, Kiro, and OpenCode.
- Mobile chat send recovery is smoother. Normal sends that race an already-active stream recover instead of leaving users at a conflict banner, while queued follow-up messages stay intact.
- Conversation reset can now repair a missing saved CLI profile by accepting an explicit valid replacement profile, instead of forcing the reset flow to fail when the stored profile was deleted.
- OpenCode with local Ollama models is easier to set up and recognize. The docs now include an Ollama/OpenCode guide, and Ollama-backed OpenCode profiles use a dedicated provider logo in settings and chat on desktop and mobile.
- CLI profile terminology is clearer. Settings now identify the underlying runtime as the harness, while chat composer selectors continue to use Profile and CLI Profile for saved run configurations.
- Claude Code Interactive has been validated with Claude Code 2.1.152, so update warnings and compatibility checks reflect the newest tested CLI boundary.

## Developer Details

- Added `docs/user/ollama-opencode.md` and linked it from the docs index, user guide, backend guide, and coverage inventory. See #369.
- Added the browser-safe `src/contracts/streamFrames.ts` contract, moved desktop stream-frame state transitions into `web/AgentCockpitWeb/src/stream/streamFrameReducer.ts`, kept `streamStore.js` as the socket/API/timer effect adapter, and recorded the boundary in ADR-0077. See #370.
- Split large desktop and mobile frontend surfaces into focused modules and CSS aggregators, including desktop shell/chat/composer surfaces and mobile app-model, lifecycle, screen, modal, icon, primitive, and style modules. See #371.
- Kept `ChatService` as the public facade while extracting focused private stores for workspace sessions, conversation lifecycle, messages, usage, Memory persistence, and Knowledge Base cache state under `src/services/chat/`. See #372.
- Added missing-native-session recovery snapshots with focused backend/session recovery stores, per-backend recovery paths, visible user notices, diagnostic metadata, and ADR-0078. See #373.
- Updated mobile send-in-flight handling so stream collision recovery preserves queue state and avoids a stale conflict state. See #374.
- Renamed the canonical CLI profile runtime identity from `vendor` to `harness` across contracts, services, settings, desktop, mobile, auth, update, instruction, migration, and import/export paths, while accepting legacy `vendor` only at compatibility boundaries. ADR-0079 records the decision. See #375.
- Added the Ollama provider icon and OpenCode/Ollama provider identity recognition across desktop Settings, desktop chat authorship, mobile app-model projection, mobile chat authorship, specs, and static coverage. See #376.
- Added an optional reset repair body for missing CLI profiles, plus route validation, desktop stream-store repair selection, mobile reset repair handling, browser-safe contracts, and focused tests. See #377.
- Added a conversation-scoped Workspace Context markdown preview/download route and desktop/mobile link interception so generated Workspace Context markdown links resolve through the file previewer instead of plain broken paths. See #379.
- Advanced the Claude Code Interactive tested CLI version to 2.1.152 after real terminal and browser compatibility validation, and updated backend/API specs plus CLI update warning fixtures. See #380 and #378.
- Updated API, backend-service, data-model, frontend, mobile PWA, Workspace Context, testing, coverage, deployment, product, README, user-doc, ADR index, and project-guidance docs for the shipped behavior.
- Mobile PWA impact was evaluated across the release. Runtime mobile behavior changed for stream recovery, reset repair, Workspace Context markdown links, Ollama provider identity, shared stream-frame contracts, and the frontend module split.
