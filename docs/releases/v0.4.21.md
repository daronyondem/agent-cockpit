# Agent Cockpit v0.4.21

## Shipped For Users

- Fixed Windows self-update so installs with a damaged or incomplete private Node.js runtime can repair the runtime and continue updating instead of failing at `npm ci`.
- Fixed the Windows welcome screen so Claude Code and Codex installed from the welcome actions are detected immediately after installation.
- Strengthened project guidance so future installer, updater, and welcome fixes stay Windows-specific and preserve the already-working macOS install/update path.

## Developer Details

- Windows production self-update now validates installer-owned private Node runtimes before reuse, including `node.exe`, `npm.cmd`, `npx.cmd`, and npm's `npm-cli.js`, `npx-cli.js`, and `npm-prefix.js` entrypoints.
- Windows private-runtime npm execution now uses `node.exe node_modules\npm\bin\npm-cli.js ...` for update-time dependency installs and build preflights, avoiding `.cmd` shim quoting for the installer-managed path.
- Windows private runtime repair now installs a sibling replacement runtime when the target runtime directory already exists, avoiding deletion of a potentially locked `node.exe`.
- Install Doctor now probes Windows Claude/Codex npm shims as `claude.cmd` and `codex.cmd`, preferring the installer-recorded `nodeRuntime.binDir` before falling back to `PATH`; macOS/Linux continue probing `claude` and `codex`.
- Added `docs/agent-project-memory.md` and linked it from `AGENTS.md` so stable project memory has a source-controlled mirror connected to canonical agent instructions.
- Updated backend/API/testing specs and focused Jest coverage for Windows runtime repair, Windows private-runtime npm invocation, and refreshed welcome detection after Claude/Codex installs.
