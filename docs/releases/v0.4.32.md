# Agent Cockpit v0.4.32

## Shipped For Users

- Fixed the Windows Codex install action on the welcome screen when the Agent Cockpit install path contains spaces. The current-user `Path` update no longer depends on PowerShell `$args` binding, so the install can finish and expose `codex` in new PowerShell or terminal windows.
- Welcome-screen Claude Code and Codex login now uses the same system CLI auth home as terminal `claude` and `codex` commands. After logging in through Agent Cockpit, new chats and terminal sessions use the same account state.
- Existing setup profiles from earlier Windows releases are migrated back to system CLI auth the next time welcome setup checks or starts login, avoiding the stale private-auth state that could say login was verified while chats still failed.

## Developer Details

- Changed the Windows user PATH helper to embed the target `cli-tools` path in the PowerShell script with single-quote escaping instead of passing it as a trailing `-Command` argument.
- Changed welcome setup-auth routing for Claude Code and Codex to create/reuse setup account profiles without `configDir`; explicit account-profile auth still uses deterministic isolated config directories.
- Added ADR-0064 and updated API/backend/data-model/testing specs, project memory, AGENTS guidance, and README. No mobile PWA behavior changed.
- macOS CLI launch and explicit account-profile isolation behavior remain unchanged.
