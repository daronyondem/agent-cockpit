# Agent Cockpit v0.4.8

## Shipped For Users

- Added a first-class Windows installer path. Windows users can now install Agent Cockpit with a PowerShell command, without admin rights, without preinstalled Node.js, and without global PM2 setup.
- Added Windows logon startup for the current user. The installer registers a current-user ONLOGON Scheduled Task, starts Agent Cockpit immediately, and opens first-run setup in the browser.
- Added Windows production self-update support. Production installs can update from GitHub Release ZIP artifacts, keep install data outside replaceable app code, and roll back automatically when the updated server fails its health check.
- Made the Windows setup path explicit: it is per-user and interactive-login based, not a Windows Service and not a no-login background service.
- Improved Codex defaults for local installs by aligning the default execution posture with Agent Cockpit's trusted local-machine model while keeping stricter deployments configurable.
- Added source-controlled release notes so GitHub Releases now include a readable shipped list and a link to detailed implementation notes.

## Developer Details

- Added `scripts/install-windows.ps1` with production and dev modes, private Node.js 22 ZIP installation, GitHub Release ZIP download and SHA256 verification, dependency installation, desktop/mobile build checks, `.env` and PM2 ecosystem generation, install-local `PM2_HOME`, helper scripts, current-user scheduled-task registration, setup polling, and repair-friendly reruns.
- Extended release packaging and `.github/workflows/release.yml` to emit and publish the Windows app ZIP plus `install-windows.ps1` while preserving existing macOS tarball and installer artifacts. The Release workflow now runs a Windows smoke job before publishing.
- Refactored production self-update so macOS keeps symlink activation while Windows activates versioned release directories through `install.json`, uses PowerShell restart/rollback scripts, handles Windows private Node ZIP runtimes, and resolves `.cmd` shims safely through `cmd.exe`.
- Hardened Windows update and installer state writes to use no-BOM UTF-8 for `install.json`, avoiding JSON parse failures under Windows PowerShell 5.1. Windows update PM2 config now preserves existing `WEB_BUILD_MODE` and `AUTH_ENABLE_LEGACY_OAUTH` values from `.env`.
- Made Install Doctor Windows-aware with Windows command resolution, scheduled-task checks, platform-appropriate remediation, and Windows-safe npm/npx action execution.
- Extended install state and public install-status types with installer-recorded Node runtime and startup metadata.
- Added ADR-0063 for the per-user Windows installer decision and updated deployment, backend services, API, data model, mobile PWA, testing, coverage, server security, and README documentation.
- Added release-note infrastructure from PR #315: `docs/release-workflow.md`, `docs/release-notes-prompt.md`, `scripts/render-release-notes.js`, release-note tests, and workflow rendering of source-controlled release documents.
- Changed Codex's default server runtime posture in PR #316 to `CODEX_APPROVAL_POLICY=never` and `CODEX_SANDBOX_MODE=danger-full-access`, with ADR-0062 and spec/test updates. Existing environment variables still allow restricted deployments.
