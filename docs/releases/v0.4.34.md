# Agent Cockpit v0.4.34

## Shipped For Users

- Fixed the remaining Windows Claude Code login mismatch where Agent Cockpit could verify a setup profile that still pointed at an old private auth directory while terminal `claude` and new chats used the normal Windows user home and stayed logged out.
- Older Windows installs now repair stale first-run Claude/Codex setup profiles automatically when settings are read or when the profile is checked, so setup login, account checks, terminal commands, and new chats use the same vendor auth state.

## Developer Details

- Added a shared `isSetupAccountCliProfile()` helper and made setup account profiles a settings-level invariant: `SettingsService` strips `configDir`, `CLAUDE_CONFIG_DIR`, and `CODEX_HOME` from `setup-claude-code-account*` and `setup-codex-account*` on save and read.
- Updated `CliProfileAuthService.profileWithAuthDefaults()` so direct checks of setup profiles remove stale auth-home fields instead of assigning deterministic private config directories. Explicit non-setup account profiles keep isolated config directories.
- Added ADR-0065 and updated API/backend/data-model/testing specs, project memory, and AGENTS guidance. No mobile PWA behavior changed.
- macOS installer/update behavior remains unchanged; focused macOS regression coverage was run before release.
