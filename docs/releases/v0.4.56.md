# Agent Cockpit v0.4.56

## Shipped For Users

- Workspace Context replaces the old Context Map experience with a simpler markdown-first workspace memory model. Users can enable it per workspace, choose a processor profile, run catch-up scans, run maintenance, inspect read-only context markdown, repair the `AGENTS.md` pointer, and clear the context folder without losing the workspace setting.
- Workspace Context now catches up after session resets and conversation archives, so durable workspace notes can be updated from completed work while reset/archive responses stay fast.
- Claude and Codex quota indicators are more reliable. Codex usage now prefers direct OAuth with refresh and falls back to bounded app-server probes, while Claude usage can fall back to CLI `/usage` parsing when the OAuth endpoint is unavailable or rate-limited.
- Claude Code Interactive supports Claude Code 2.1.143. The hidden terminal flow accepts the newer bypass-permissions warning, handles prompt readiness when hooks are unavailable, and avoids leaving browser streams marked active after terminal messages arrive.
- The README and public docs now provide clearer product fit, quickstarts, backend guidance, user guides, deployment guides, and reference pages for local storage, environment variables, development, and testing.
- Historical design, research, and archived Context Map materials were moved out of the primary spec path, making the docs index easier to navigate while preserving records that are still useful.

## Developer Details

- Added ADR-0068 and `docs/spec-workspace-context.md` for the accepted Workspace Context design. The new design keeps canonical context in `data/chat/workspaces/{hash}/workspace-context/context/`, records short-lived run reports under `runs/`, and uses a managed workspace `AGENTS.md` block that points provider CLIs at `WORKSPACE_CONTEXT.md`.
- Removed the Context Map graph subsystem, candidate review pipeline, SQLite store, MCP surface, report script, routes, contracts, and tests. The obsolete spec stub now lives under `docs/archive/context-map-spec.md`.
- Added `src/services/workspaceContext/service.ts`, `src/routes/chat/workspaceContextRoutes.ts`, `src/contracts/workspaceContext.ts`, Workspace Context stream/status events, settings migrations from legacy Context Map fields, run-state tracking, scheduled scans, maintenance runs, finalizer integration for session reset/archive, and focused service/API tests.
- Updated the desktop Workspace Settings surface to configure Workspace Context processor defaults or overrides, run scans and maintenance, preview markdown files, show run history, repair workspace instructions, and clear Workspace Context data. The composer notification now reflects active or failed Workspace Context runs.
- Hardened Codex plan usage collection by refreshing stale direct OAuth tokens before app-server fallback, running read-only bounded fallback probes, recovering usage payloads from RPC decoder errors, and normalizing session/weekly windows into stable slots.
- Hardened Claude plan usage collection by adding a CLI `/usage` fallback, retaining quota source/identity diagnostics, parsing collapsed terminal labels and delayed bars, handling unknown/null buckets and scope errors, and exposing source/identity details in the Claude usage tooltip.
- Updated Claude Code Interactive compatibility for CLI 2.1.143, including bypass-permissions warning handling, prompt-readiness fallback behavior, stream finalization, version constants, and real-CLI compatibility coverage for terminal and browser flows. Re #326.
- Added public documentation layers under `docs/user/`, `docs/deploy/`, and `docs/reference/`, rewrote the README around the local-first product positioning, and added `docs/product-positioning.md` for future public copy.
- Reorganized implemented plans into `docs/design/`, empirical notes into `docs/research/`, and historical Context Map material into `docs/archive/`. Accepted ADR link updates were made as a one-time exception during the docs reorganization.
- Mobile PWA impact was evaluated. No mobile runtime files changed; Workspace Context work ran mobile typecheck/build because shared workspace contracts and settings changed.
