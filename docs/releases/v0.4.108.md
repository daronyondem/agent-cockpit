# Agent Cockpit v0.4.108

## Shipped For Users

- Knowledge Base search can now repair its vector index from the KB browser. When embedding search coverage looks stale or broken, users can rebuild the derived vector index without re-uploading files or rerunning Dream/Re-Dream.
- Knowledge Base vector rebuilds are more resilient. If the existing PGLite vector store is corrupt or cannot open, Agent Cockpit resets only that derived vector directory and rebuilds entries and topics from the preserved SQLite KB state.
- Mobile chat send recovery is smoother when the server accepts a turn but the request still fails locally. The PWA keeps the optimistic user message visible, leaves the composer clear, and reconciles with the server conversation instead of rolling back a successful send.
- Claude Code profiles now expose Claude Opus 4.8 with image-capable model metadata, Opus effort levels, and built-in usage pricing.
- Claude Code Interactive has been validated with Claude Code 2.1.154, so compatibility warnings and update checks reflect the newest tested CLI boundary.
- Agent Cockpit-owned system notices now use the Agent Cockpit identity consistently in desktop chat, mobile chat, read-only mobile sessions, and pinned-message source labels instead of inheriting the active CLI profile identity.

## Developer Details

- Added `POST /api/chat/workspaces/:workspaceId/kb/vector-index/rebuild`, backed by `src/services/knowledgeBase/vectorMaintenance.ts`, to embed all current SQLite KB entries/topics in 50-item batches, upsert them into PGLite, remove stale vector rows, broadcast `kb_state_update`, and report rebuild counts. See #381.
- Added `WorkspaceKnowledgeStore.resetVectorStore()` and a `ChatService` facade method so vector maintenance can close/evict cached handles and delete only `knowledge/vectors/` when the derived vector store cannot open. The rebuild path then recreates the store from SQLite entries/topics without mutating raw, converted, entry, synthesis, or Dream state. See #383.
- Updated the KB browser maintenance UI and API client to surface the rebuild action from degraded vector-search states, with specs covering endpoint behavior, backend service responsibilities, frontend behavior, and focused test coverage.
- Updated mobile send recovery in `mobile/AgentCockpitPWA/src/App.tsx` and `appModel.ts` to distinguish true active-stream conflicts from accepted sends that only need server reconciliation. Mobile specs and static/model tests cover the new optimistic-message behavior. See #384.
- Added `claude-opus-4-8` to Claude Code backend metadata, updated model examples and data-model docs, added Anthropic pricing catalog coverage for the Opus 4.8 family, and advanced `CLAUDE_CODE_INTERACTIVE_TESTED_CLI_VERSION` to `2.1.154` after real CLI/browser validation. See #385 and #382.
- Added browser-safe desktop message identity helpers in `web/AgentCockpitWeb/src/chat/messageIdentity.js` and matching mobile logic so `system` messages without goal events render as Agent Cockpit across chat bubbles and pin strips. See #386.
- Updated API, backend-service, data-model, frontend, mobile PWA, testing, and project-memory docs for the shipped behavior.
- Mobile PWA impact was evaluated across the release. Runtime mobile behavior changed for accepted-send recovery and system-message identity; Opus 4.8 appears through dynamic backend metadata without mobile source changes.
