# Agent Cockpit v0.4.39

## Shipped For Users

- Pandoc and LibreOffice can now be installed while Agent Cockpit is already running, then detected from the Welcome screen by clicking **Refresh**. A restart is no longer required just to update those optional-tool checks.
- Knowledge Base setup now reacts to newly installed document-conversion tools immediately: DOCX upload checks refresh Pandoc before rejecting a file, and PPTX slide-image conversion refreshes LibreOffice before falling back.
- Windows optional-tool detection is more forgiving after installer-based installs. Agent Cockpit checks the current process PATH, current Windows user and machine PATH values, and common Pandoc/LibreOffice install locations.

## Developer Details

- Added a shared external-tool detection helper for Knowledge Base dependencies. Windows probes merge live environment PATH with `HKCU\Environment\Path` and `HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Environment\Path`, expand `%VAR%` entries, hide spawned console windows, and fall back to known Pandoc/LibreOffice install directories.
- Extended `detectPandoc()` and `detectLibreOffice()` with a `{ refresh: true }` option. User-facing status paths force refresh while preserving module-level cached results for normal runtime calls.
- Updated Install Doctor to refresh Pandoc/LibreOffice on every status request and changed remediation text from restart guidance to refresh guidance.
- Updated `GET /kb/pandoc-status`, `GET /kb/libreoffice-status`, DOCX upload preflight, and PPTX rasterization to refresh tool detection before reporting missing dependencies.
- Updated Welcome, Knowledge Base UI copy, backend/service specs, API specs, frontend specs, test specs, and project memory to document refresh-based optional-tool detection.
- Added focused Install Doctor coverage proving repeated doctor status calls re-probe Pandoc and LibreOffice and can transition from warning to ok without a server restart.
- Mobile PWA impact was evaluated. No mobile files changed because this behavior is limited to desktop Welcome, Install Doctor, and Knowledge Base dependency checks.
