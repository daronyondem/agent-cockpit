# Agent Cockpit v0.4.33

## Shipped For Users

- Fixed a Claude Code login false-positive where Agent Cockpit could show a welcome/account login as verified even though Claude Code still reported `loggedIn: false` and chats or terminal `claude` commands remained unauthenticated.
- Claude Code login jobs now keep verifying until `claude auth status --json` reports `loggedIn: true`; if the CLI stays logged out, Agent Cockpit shows the failure instead of marking the login complete.

## Developer Details

- Added vendor-specific auth status interpretation in `CliProfileAuthService`: Codex remains exit-code based, while Claude Code now parses status JSON and rejects logged-out exit-0 responses.
- Added focused regressions for Claude status checks and post-login verification when the CLI returns `{"loggedIn":false}` with exit code `0`.
- Updated API/backend/testing specs and project memory. No mobile PWA behavior changed.
