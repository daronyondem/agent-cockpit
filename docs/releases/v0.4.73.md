# Agent Cockpit v0.4.73

## Shipped For Users

- Chat Markdown downloads now let users choose whether to export the current session or all sessions for a conversation, with matching controls in the desktop web UI and mobile PWA.
- Usage views now show provider-aware costs and estimated costs by backend, model, and service tier. Provider-reported costs are preserved when available, while built-in pricing and user overrides fill gaps for historical and current usage.
- Agent Cockpit can now be installed and updated on Ubuntu 24.04 LTS x64/glibc using the production Linux installer, release assets, manifest entries, checksums, and smoke-tested update path.
- Codex goal status updates now appear live during normal chat streams, so users no longer need to refresh the page to see a goal that Codex started or cleared during the current turn.
- Worktree isolation lets each conversation run in its own Git worktree and session branch, reducing file-edit collisions when multiple conversations are active in the same repository.
- Workspace identity is now stable even when a workspace folder moves. Existing conversations, settings, Memory, Knowledge Base data, Workspace Context data, and Git features follow the immutable workspace ID, and the web UI can remap a workspace location.
- Global Settings now includes full data-root export/import for migration and backup. Exports use `.acexport` bundles with manifests and checksums, while imports stage replacement, preserve backups, and run post-import checks.
- The mobile PWA now keeps the selected CLI profile visible during the first live response in a new stream and renders multi-paragraph thinking blocks as readable full-width panels on narrow screens.
- Claude Code Interactive now supports Claude Code 2.1.148, and the compatibility monitor keeps newer-version support issues focused on the latest open validation target.
- Windows startup and CLI detection are more reliable for manual development installs and npm shim layouts, including Windows-safe build preflight command selection and package-entrypoint resolution near PATH-visible Claude/Codex shims.

- Kept Windows manual startup builds on `npm.cmd`, improved Windows Claude/Codex command resolution near npm shims, and documented fresh-install CLI profile guidance while preserving non-Windows build paths. See #341.
- Added per-conversation Git worktree isolation with shared contracts, worktree routes, service ownership under `src/services/chat/`, dirty-check blockers, enable/disable migration, reset handling, upload/Git/status/workspace execution-directory routing, ADR-0072, user docs, and focused tests. See #343 and #342.

See #345.
- Added data-root `.acexport` migration with manifest checksum validation, staged destructive replacement after exact confirmation, backup preservation, startup apply, post-import checks, migration routes, desktop Settings UI, ADR-0074, and focused route/service tests.
