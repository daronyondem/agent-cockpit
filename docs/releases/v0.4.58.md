# Agent Cockpit v0.4.58

## Shipped For Users

- Claude Code Interactive now supports Claude Code 2.1.145, so users with the current Claude CLI can use Interactive mode without the newer-version compatibility warning.
- Plan approval flows in Claude Code Interactive are more reliable after browser reconnects or stream replays. Once a final response arrives, stale replayed plan prompts no longer keep the composer locked in "answer the prompt above" mode.

## Developer Details

- Advanced `CLAUDE_CODE_INTERACTIVE_TESTED_CLI_VERSION` from `2.1.143` to `2.1.145` after passing the real terminal and browser compatibility suites. Re #332.
- Updated desktop `StreamStore` handling so final response messages clear stale pending plan/question interaction state and recompute `uiState`, covering the replay order observed while validating Claude Code 2.1.145.
- Added `test/streamStore.test.ts` regression coverage for a replayed plan approval followed by a final response message and `done`.
- Updated CLI update compatibility expectations and specs for the new tested Claude Code Interactive version and replay-state lifecycle.
- Closed the superseded Claude Code 2.1.144 validation issue in favor of the newer Claude Code 2.1.145 validation issue. Re #331.
- Mobile PWA impact was evaluated. No mobile runtime files changed; this release touches desktop stream state, Claude Code Interactive compatibility metadata, and specs/tests.
