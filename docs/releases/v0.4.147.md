# Agent Cockpit v0.4.147

## Shipped For Users

- Usage Stats now include Claude Code usage from terminal sessions that were not started inside Agent Cockpit. Imported transcript usage is checkpointed so later stats refreshes do not double-count the same session.
- Claude Code conversations now support an Ultracode mode on desktop and mobile. The selected mode is persisted with conversations, sends, goals, and active stream metadata so follow-up turns keep the intended Claude Code behavior.
- Claude Code Interactive has been validated with Claude Code 2.1.172, so compatibility warnings and update checks reflect the newest tested CLI boundary.
- Release reliability is stronger. The repository now has a zero-warning lint gate, type-aware promise checks, coverage thresholds in CI, stabilized parallel Jest timing, and documented audit handling before production release packaging.

## Developer Details

- Added `ClaudeTranscriptUsageImportService` and ledger checkpointing so Usage Stats can import external Claude Code transcript usage without duplicating Agent Cockpit-owned sessions or previously imported rows. API/data-model/backend-service specs and focused tests were updated. See #418.
- Added the `claudeCodeMode: "ultracode"` contract, persistence, desktop/mobile controls, standard Claude Code `--settings` wiring, Claude Code Interactive hook settings merging, ADR-0085, and focused adapter/client tests. See #419.
- Remediated the non-force `qs` audit path by moving Express/body-parser dependencies forward and documented the remaining PM2 nested `ws` exposure under #346. The root audit still exits non-zero for that accepted PM2 advisory; the mobile audit is clean. See #421.
- Removed the retired `public/v2/` placeholder tree, archived completed root planning docs under `docs/archive/`, added retired-prefix handling for ADR lint, and updated specs/coverage indexes. See #420 and #422.
- Added the root ESLint flat config, type-aware promise correctness rules, React Hooks checks, CI/release workflow lint gates, and ADR-0087. A follow-up burned down the warning baseline and made `npm run lint` a zero-warning gate. See #423 and #443.
- Added backend service coverage for stream jobs, local auth, CLI profile auth, session finalization, and Claude plan usage; then introduced V8 coverage thresholds and a parallel CI coverage job with the measured baseline in `jest.config.js`. See #424 and #425.
- Split server-only type declarations from `src/types/index.ts` into declaration-only domain modules behind the existing barrel, with a maintainability check and ADR-0088 documenting the boundary. See #429.
- Extracted ChatService internals into focused modules under `src/services/chat/` for memory search/metadata, legacy migrations, session transitions, transcript markdown, Workspace Context status, KB snapshots, conversation policy, and worktree-isolation toggles while keeping `ChatService` as the facade. See #426.
- Split the Knowledge Base SQLite implementation into focused modules under `src/services/knowledgeBase/db/` behind the existing `KbDatabase` facade. See #427.
- Split the Codex backend into runtime, protocol, event, stream-dispatch, artifact, usage, model, exec, and process modules while preserving `src/services/backends/codex.ts` as the public facade and compatibility re-export surface. ADR-0089 records the shared dispatcher boundary. See #428.
- Refactored the mobile PWA shell so `App.tsx` composes focused hooks for stream connection, send pipeline, goal state, list monitoring, attachments, queue editing, run selection, session history, file preview, and workspace explorer behavior. Added pure mobile stream-event projection tests. See #430.
- Stabilized parallel Jest timing with capped worker fan-out and wider timeout headroom for route/process-heavy suites, then advanced Claude Code Interactive compatibility to 2.1.172 after real terminal and browser reports passed. See #442 and #445.
