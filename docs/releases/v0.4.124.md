# Agent Cockpit v0.4.124

## Shipped For Users

- iPhone RAW `.dng` chat uploads now work as image attachments. Agent Cockpit extracts the embedded JPEG preview, caps oversized previews to the existing vision-safe image size, and passes the generated `.dng.preview.jpg` to the selected CLI while keeping the original DNG in the conversation artifacts.
- Chat transcripts stay pinned to the latest useful activity more reliably on desktop and mobile. Tool activity, thinking updates, outcomes, and artifact-only changes now retrigger auto-follow when the user has not intentionally scrolled away.
- The mobile PWA recovers more gracefully when Safari/WebKit rejects WebSocket construction. Active chat streams and passive conversation monitors fall back to the existing polling paths instead of leaving the UI disconnected.
- OpenCode usage accounting is more complete for providers that omit live usage frames. Agent Cockpit now recovers provider-reported usage from exported OpenCode sessions without double-counting providers that already stream usage.
- OpenCode profiles using OpenRouter are easier to recognize. Desktop and mobile now show the OpenRouter icon for OpenRouter-backed OpenCode profiles and hide the technical `openrouter/` prefix from user-facing model labels while preserving the raw model id for execution.
- Claude Code Interactive has been validated with Claude Code 2.1.161, so compatibility warnings and update checks reflect the newest tested CLI boundary.

## Developer Details

- Added DNG preview extraction behind chat upload normalization. `dngPreview.ts` parses bounded TIFF/DNG IFDs, selects the largest readable single-strip JPEG preview, skips raw subimages, resizes previews above a 2576 px long edge through `@napi-rs/canvas`, and returns a generated sibling image attachment from `uploadRoutes.ts`. ADR-0083 records the cross-platform preview-extraction decision. See #399.
- Raised conversation upload limits to 100 MB per file, added DNG-specific malformed/no-preview `400` responses, and best-effort paired original-DNG cleanup when deleting generated preview attachments. API, backend-service, data-model, frontend, mobile PWA, testing, and coverage specs now describe the route behavior. See #399.
- Added a shared rendered-block scroll signature for desktop and mobile chat. Assistant activity, thinking, tool outcomes, and artifacts now update the transcript scroll key even when message text is unchanged, with focused model and route coverage. See #398.
- Treated synchronous mobile WebSocket constructor failures as recoverable transport misses. Mobile chat streams rely on `/api/chat/active-streams` polling after constructor rejection, and passive list monitors remain best-effort. See #397.
- Updated OpenCode stream finalization to recover usage from exported session data when live JSON usage frames are absent, preserving the existing live-frame path and preventing duplicate usage rows. Focused OpenCode adapter tests cover recovery and non-duplication. See #403.
- Added OpenRouter identity metadata for OpenCode profiles, including `public/icons/openrouter-logo.svg`, desktop/mobile icon selection, assistant-avatar styling, and model-label formatting that hides only the display prefix. See #405.
- Advanced `CLAUDE_CODE_INTERACTIVE_TESTED_CLI_VERSION` first to 2.1.159 and then to 2.1.161 after real terminal and browser compatibility reports passed. Backend/API specs and CLI update warning coverage were kept aligned. See #401 and #406.
- Updated frontend, mobile PWA, backend-service, API endpoint, data-model, testing, coverage, and ADR docs for the shipped behavior.
- Mobile PWA impact was evaluated across the release. Runtime mobile behavior changed for WebSocket constructor recovery, transcript auto-follow, DNG upload handling through the shared upload endpoint, and OpenRouter-backed OpenCode profile display.
