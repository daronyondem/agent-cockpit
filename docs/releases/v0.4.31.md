# Agent Cockpit v0.4.31

## Shipped For Users

- Windows Claude Code and Codex installs from the welcome screen add Agent Cockpit's per-user `cli-tools` directory to the current user's Windows `Path`, so new PowerShell or terminal windows can run `claude` and `codex`.
- Existing Windows installs that update through Agent Cockpit also repair that user `Path` entry, so users do not need to rerun the full installer just to expose already-installed CLIs in terminals.
- Windows detection recognizes Claude Code and Codex CLIs that users installed themselves outside Agent Cockpit, including PATH-visible `.exe` and `.cmd` commands.

## Developer Details

- Added a Windows user PATH helper that prepends the install-local `cli-tools` directory to the live server environment, writes it to the current user's `Path` with PowerShell, and broadcasts an environment-change notification.
- Wired the helper into welcome-screen CLI install actions, the Windows installer repair path, and Windows production self-update.
- Extended the shared Windows CLI resolver to check PATH-hosted `claude.exe`/`claude.cmd` and `codex.exe`/`codex.cmd` fallbacks after managed package entrypoints.
- Updated Install Doctor, UpdateService, the Windows installer script, specs, project memory, AGENTS guidance, README, and focused tests. No mobile PWA behavior changed.
