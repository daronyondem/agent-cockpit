# Agent Cockpit v0.4.49

## Shipped For Users

- Long desktop chat transcripts now stay responsive by loading the visible session in bounded message windows. Typing, switching conversations, loading older history, jumping to pinned messages, and returning to the live tail are smoother without adding a virtual scrolling dependency.
- The mobile PWA now shows sent messages immediately and displays a live response placeholder while the send request starts, so mobile chat feels responsive even before the server acknowledges the message.
- New conversations now receive generated titles on their first session, and sidebar titles stay in sync even when title updates arrive while the conversation list is still loading.
- CLI update status is more reliable after Welcome creates or installs Claude/Codex profiles. Unprobed rows are refreshed automatically, disabled update actions look disabled, and the no-update footer is clearer.
- Windows CLI setup is more tolerant when persisting the current user's PATH fails after npm succeeds: wrapper repair and refreshed detection continue, and the generated PowerShell PATH helper avoids the Windows PowerShell `TrimEnd` failure.
- Background memory extraction after a session reset now refreshes Memory views silently instead of adding a synthetic Memory bubble to archived chat transcripts. Explicit live `memory_note` writes still show in the originating chat.
- The desktop Files Browser aligns file sizes consistently and reuses that area for row actions on hover or focus, keeping rows easier to scan.

## Developer Details

- Added ADR-0067 for dependency-free desktop transcript paging. `GET /conversations/:id` now supports `messageWindow=tail&limit=N`, and `GET /conversations/:id/messages` returns tail, older-than-anchor, and around-anchor windows with `messageWindow` indexes plus `pinnedMessages`.
- Updated desktop `StreamStore` and `ChatLive` to request a tail page, prepend older pages, cap mounted transcript rows, load around pinned messages when needed, restore the tail for "Back to end", memoize transcript rendering, and preserve per-conversation feed positions.
- Kept the full conversation read contract backward-compatible for callers that do not request a message window. The server still reads the active session file before slicing, so this release primarily reduces browser rendering cost.
- Removed the browser-side `goalState.js` CommonJS fallback and adjusted Jest coverage to load it through the existing transform, eliminating the Vite/Rolldown CommonJS warning during web builds.
- Fixed chat title generation by requesting title generation for brand-new first-session conversations and by queuing sidebar `title_updated` patches until the conversation list contains the row.
- Added the web-only CLI update store freshness path for unprobed rows and covered Settings/composer update-state behavior in `test/cliUpdateStore.test.ts` and `test/frontendRoutes.test.ts`.
- Added mobile model helpers for optimistic transcript replacement/removal and updated Mobile PWA tests/specs for pending user bubbles and live response placeholders.
- Changed post-session Memory extraction to emit `sourceConversationId:null` and `displayInChat:false`, while preserving conversation-visible `memory_note` write outcomes.
- Fixed the Windows user PATH PowerShell generator to normalize entries with a regex instead of a multi-character `TrimEnd` argument, and kept Install Doctor CLI wrapper repair/detection running when PATH persistence is only a warning.
- Stabilized local Jest HTTP/MCP tests by binding helper servers to `127.0.0.1`, adding bounded request timeouts, and lazy-loading Knowledge Base format handlers so unrelated route tests do not import native PDF/canvas modules at startup.
- Updated API, data-model, backend-service, frontend, mobile PWA, testing, ADR, and project-memory docs for the changed behavior.
- Mobile PWA impact was implemented directly for optimistic sends. Other desktop/backend fixes were evaluated and documented as not changing mobile behavior.
