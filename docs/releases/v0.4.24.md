# Agent Cockpit v0.4.24

## Shipped For Users

- Fixed Windows reinstall and repair runs so PM2 cleanup no longer prints npm `npx canceled due to missing packages` errors before dependencies have been installed.
- Fixed the Windows welcome diagnostics path so PM2 is checked through the installed local command instead of asking npm to resolve PM2 interactively.

## Developer Details

- `scripts/install-windows.ps1` now resolves `<appDir>\node_modules\.bin\pm2.cmd` directly for PM2 cleanup, startup, save, and diagnostics. Best-effort cleanup skips when local PM2 is not installed yet, which is expected for freshly extracted or partially repaired app directories.
- `InstallDoctorService` now uses app-local `pm2.cmd --version` on Windows while preserving the existing non-Windows `npx --no-install pm2 --version` behavior.
- Added focused Windows installer and install-doctor coverage, and updated deployment/API/backend/testing specs to document the Windows-specific PM2 command path.
