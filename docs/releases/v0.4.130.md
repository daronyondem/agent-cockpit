# Agent Cockpit v0.4.130

## Shipped For Users

- Workspace Routines add repeatable workspace workflows. Enable Routines per workspace, install proposals from chat, run them manually, schedule recurring runs, inspect output and persistent state, and send Telegram notifications when a routine writes a notification.
- Routines are opt-in by default. Disabled workspaces do not install managed workspace instructions, scaffold routine files from passive reads, or run schedules. Turning Routines off removes only the managed instruction block and keeps routine data intact.
- Attachment OCR now follows the selected CLI profile and backend on desktop and mobile. If a conversation is missing its stored profile reference, OCR can repair it and adds a visible Agent Cockpit recovery notice instead of failing silently.
- Claude Code Interactive compatibility is validated through Claude Code CLI 2.1.170, keeping update warnings aligned with the current tested boundary.
- Claude Fable 5 is available in Claude Code model selectors with image-capable metadata, effort controls, and built-in usage-cost estimates.

## Developer Details

- Added the Workspace Routines infrastructure under `src/services/routines/`, browser-safe contracts in `src/contracts/routines.ts`, REST routes in `src/routes/chat/routineRoutes.ts`, scheduler support, run folders, routine output and persistent-state browsing, proposal cards, desktop settings UI, Telegram destination connection, and user documentation.
- Added ADR-0084 and the canonical Routines spec covering manifest shape, proposal markers, opt-in workspace gating, run records, schedule behavior, file layout, retention fields, notification delivery, and desktop UI behavior.
- Introduced per-workspace `routinesEnabled` state and gating so disabled workspaces preserve existing routine data while blocking list/action/scheduler/proposal entrypoints and removing stale managed workspace instruction blocks.
- Updated OCR upload contracts, desktop clients, mobile clients, and `/conversations/:id/attachments/ocr` handling so the selected runtime is passed through, one-shot image support is validated, model image capability is respected, locked-session switches are rejected, and profile recovery messages are persisted.
- Advanced `CLAUDE_CODE_INTERACTIVE_TESTED_CLI_VERSION` to `2.1.170` and updated compatibility warning coverage and specs after real terminal and UI validation.
- Added `claude-fable-5` metadata in the Claude Code backend catalog, Anthropic pricing catalog entry `2026-06-09`, model/pricing coverage, shared type comments, and backend/API/data-model/testing specs.
