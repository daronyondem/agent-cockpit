# Agent Cockpit v0.4.90

## Shipped For Users

- OpenCode is now a supported backend. Users can add self-configured OpenCode profiles and run providers such as DeepSeek, Groq, Gemini, and OpenRouter from the same desktop and mobile chat surfaces used for Claude Code, Codex, and Kiro.
- OpenCode profiles can discover provider-specific model catalogs, including reasoning-effort variants and media input support, so model choices in the composer and mobile Run Settings reflect the selected provider.
- OpenCode conversations stream visible reasoning, text, usage, session continuity, and tool activity into Agent Cockpit. Later turns resume through persisted OpenCode session ids.
- Settings can check OpenCode profile drafts before saving, verify installed OpenCode CLIs, and run OpenCode's self-update command from the CLI update surface.
- OpenCode provider identity is clearer in chat. Messages from OpenCode profiles can display the selected provider, such as DeepSeek, while the backend remains OpenCode for runtime and transcript compatibility.
- Image and PDF one-shot workflows now check both backend transport and selected-model media support, so attachment OCR and Knowledge Base media conversion avoid sending unsupported files to text-only models.

## Developer Details

- Added ADR-0076 for OpenCode CLI profiles. OpenCode is modeled as the physical CLI vendor, while DeepSeek, Groq, Gemini, OpenRouter, and similar APIs remain provider/model choices owned by OpenCode rather than top-level Agent Cockpit vendors. See #368.
- Added `OpenCodeAdapter` with `opencode run --format json`, visible-thinking mapping, tool activity/outcome mapping, usage parsing, persisted `external_session` ids, later-turn resume with `--session`, one-shot execution, title/summary generation fallbacks, OpenCode export-based final-text recovery, and profile-aware runtime resolution.
- Added OpenCode model discovery through `opencode models [provider] --verbose` with a plain-command fallback, parsing provider/model ids, effort variants, and input/output media capability metadata.
- Added per-process Memory and Knowledge Base MCP injection for OpenCode through `OPENCODE_CONFIG_CONTENT` so scoped MCP tokens are not written into global OpenCode configuration.
- Added shared one-shot media capability checks and updated Claude Code, Codex, Kiro, and OpenCode metadata so media workflows require both backend transport support and selected-model input modality support.
- Added OpenCode CLI profile contracts, settings normalization, draft metadata/check routes, saved-profile status checks, CLI update target detection, and `opencode upgrade` support.
- Updated desktop Settings, backend selection, composer metadata, provider labels, provider avatars, and theme-aware OpenCode icons. DeepSeek has a recognized provider icon.
- Updated the mobile PWA Run Settings and message author rendering so provider-scoped OpenCode metadata and provider identity match desktop behavior. No native mobile path changes were needed.
- Updated public docs, user docs, backend capability references, API/backend/data/frontend/mobile/testing specs, onboarding, product positioning, and ADR index entries for the new OpenCode profile path.
- Mobile PWA impact was evaluated in #368. The release changes mobile Run Settings and mobile message author display for OpenCode profiles; other mobile behavior is unchanged.
