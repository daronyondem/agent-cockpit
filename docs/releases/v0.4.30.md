# Agent Cockpit v0.4.30

## Shipped For Users

- Windows Claude Code and Codex installs from the welcome screen now add Agent Cockpit's per-user `cli-tools` directory to the current user's Windows `Path`, so new PowerShell or terminal windows can run `claude` and `codex`.
- Windows detection now also recognizes Claude Code and Codex CLIs that users installed themselves outside Agent Cockpit, including PATH-visible `.exe` and `.cmd` commands.
- The Windows installer repair path also persists the `cli-tools` PATH entry, while macOS CLI launch behavior remains unchanged.

## Developer Details

- Added a Windows user PATH helper that prepends the install-local `cli-tools` directory to the live server environment, writes it to the current user's `Path` with PowerShell, and broadcasts an environment-change notification.
- Extended the shared Windows CLI resolver to check PATH-hosted `claude.exe`/`claude.cmd` and `codex.exe`/`codex.cmd` fallbacks after managed package entrypoints.
- Updated Install Doctor, the Windows installer script, specs, project memory, AGENTS guidance, README, and focused tests. No mobile PWA behavior changed.
