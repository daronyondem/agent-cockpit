# Agent Cockpit v0.4.88

## Shipped For Users

- macOS and Linux installs now register Agent Cockpit to start automatically for the current user session by default. macOS uses a LaunchAgent, Linux uses a systemd user unit, and both installers keep a `--no-auto-start` opt-out for manual startup.
- Production self-updates on macOS and Linux are more reliable. Restart scripts are repaired when needed, launched through `sh`, and checked against the expected `/api/chat/version` response before an update is accepted as healthy.
- Fresh setup and Install Doctor now find standard macOS `LibreOffice.app` installs even when `soffice` is not on `PATH`, and can offer a PATH repair action for detected app installs.
- CLI profile recovery is clearer when a conversation points at a deleted profile. Agent Cockpit can repair the conversation when there is one enabled replacement, shows explicit errors when recovery is ambiguous, and keeps stale profile IDs out of the composer.
- Claude and Codex command discovery now covers common user-local install directories on macOS and Linux, and installer-started server processes include those directories while preserving the private Node runtime path.
- Codex one-shot responses and Memory processing are more reliable in workspaces with conversation-specific working directories.
- Mobile PWA login returns now preserve the requested in-app destination through password, passkey, and recovery-code login while rejecting unsafe external redirects.
- Knowledge Base raw-file rows now include a Download action so users can retrieve uploaded source files directly from the Raw tab.
- Data export/import now preserves PGLite Knowledge Base vector directories, including required empty runtime directories, and reports incomplete imported vector stores during post-import checks.
- Claude Code Interactive compatibility has been validated through Claude Code 2.1.150.

## Developer Details

- Added macOS LibreOffice app-bundle detection and an Install Doctor/Welcome PATH repair action, with focused service, frontend-route, and spec coverage. See #355.
- Updated CLI profile deletion and recovery flow across Global Settings, reset handling, message and goal routes, composer state projection, and stale-profile display. The same change added user-local CLI discovery for macOS/Linux, installer PATH construction updates, Codex `runOneShot()` final-message file handling, Memory MCP workspace directory propagation, default workspace branding updates, and hermetic install-doctor tests. See #357.
- Advanced Claude Code Interactive tested CLI support first to 2.1.149 and then to 2.1.150 after real terminal and browser compatibility validation, keeping update-warning fixtures and backend/API specs aligned. See #358, #365, #356, and #363.
- Hardened POSIX production self-update restart behavior by chmodding restart scripts after writes, launching restart handoff through `sh`, repairing stale restart scripts during service startup and installer repair runs, and requiring target-version health checks for production update restarts. See #359, #360, and #361.
- Preserved safe relative browser return targets through mobile login redirects and added auth coverage for password, passkey, and recovery-code flows while continuing to reject unsafe external redirects. See #362.
- Added current-user startup registration for POSIX installers with macOS LaunchAgent and Linux systemd user-unit paths, `--no-auto-start`, persisted startup ownership in `install.json`, Install Doctor startup checks, release smoke cleanup, deployment/spec docs, and ADR-0075. See #364.
- Added Knowledge Base raw-file download controls using the existing raw-file streaming route, plus frontend static coverage and UI specs. See #366.
- Updated data migration export/import to preserve PGLite vector runtime directory structure, recreate required directories while staging older imports, and flag incomplete PGLite vector stores in post-import checks. See #367.
- Made Install Doctor PATH handling and macOS startup coverage independent of the host runner platform so Windows release smoke can validate POSIX-mode checks without path-delimiter or UID-probe drift.
- Updated deployment, backend-service, API, frontend, mobile PWA, server-security, data-model, testing, release-workflow, onboarding, README, ADR index, and project-guidance docs to match the shipped behavior.
- Mobile PWA impact was evaluated across the release. Runtime mobile behavior changed for safe auth return targets; other changes are backend, installer, desktop Settings, Knowledge Base desktop UI, release workflow, documentation, or compatibility metadata work.
